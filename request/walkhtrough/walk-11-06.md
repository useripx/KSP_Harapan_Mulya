## Sesi 1: Perbaikan Template Animasi 404 & Integrasi Routing

Hari ini, pengembangan dimulai dengan sebuah laporan bug yang diajukan oleh pengguna terkait template halaman *404 Not Found*. Pengguna mengeluhkan bahwa susunan kotak (grid) dari wordsearch puzzle tidak simetris (jatuh ke baris bawah) dan animasi hurufnya mati/tidak mau menyala secara berurutan seperti yang ditunjukkan pada tangkapan layar awal.

**Langkah Perbaikan Animasi & CSS:**

- Pertama, kami menginspeksi file `index.html`, `style.css`, dan `script.js` dari direktori `request/404/dist/`.
- Terlihat bahwa masalah *wrapping* (grid jatuh) disebabkan oleh sisa perhitungan persentase margin di browser. Kami menyelesaikannya dengan memasukkan *pseudo-class* CSS `#wordsearch ul li:nth-child(8n) { margin-right: 0; }` agar kotak ke-8 pada setiap baris menempel pas di sisi kanan.
- Untuk animasi, kami menyadari *script.js* menggunakan metode lama dari jQuery yaitu `$(this).delay(1500).queue(...)` pada tingkat `$(function())`, sehingga variabel `this` mengarah ke elemen *Document*, bukan elemen *li*. Kami mengubah logikanya dengan menggunakan *loop* `setTimeout` iteratif yang terakumulasi. Huruf-huruf langsung menyala kembali dengan warna hijau (*teal*) yang cantik.
- Kami juga memperbaiki URL jQuery agar menggunakan HTTPS spesifik, sehingga dapat merender animasi meski tidak di *host* pada server lokal.

**Pemindahan ke Sistem MVC:**
Setelah template dianggap prima, pengguna meminta agar desain tersebut diimplementasikan secara resmi untuk menangani semua Error 404 pada Koperasi.

- Kami menyalin file statis menjadi aset publik: `public/assets/css/404.css` dan `public/assets/js/404.js`.
- File HTML diubah menjadi View PHP (`views/errors/404.php`), lalu kami menyematkan fungsi `asset()` agar framework mengenali letak file dengan benar.
- *Router* `public/index.php` diperbarui agar fungsi `setNotFound()` tidak lagi melakukan *echo* teks kasar, melainkan me-`require_once` tampilan estetik 404 kita.
- Tombol "Beranda" kami ubah agar melempar pengguna ke `url('/dashboard')`, yang secara elegan akan menangani redereksi sesuai tingkat wewenang (Role) pengguna secara dinamis.

## Sesi 2: Rencana Arsitektur Keamanan & Clean URL

Pekerjaan kemudian beralih ke analisis sebuah dokumen rahasia PDF bertajuk "Membuat URL Bersih dan Tersembunyi". Tujuan dokumen ini adalah memastikan bahwa struktur folder fisik server (seperti `/app`, `/views`) disembunyikan total di mata publik dan URL tidak memiliki jejak `/public/`.

Meskipun sistem Koperasi sudah menerapkan struktur *Front Controller* yang disarankan dokumen tersebut, kami menemukan satu celah fatal: **Tidak ada file `.htaccess` di *root* direktori proyek**. Hal ini memungkinkan seseorang untuk mengakses direktori internal jika dokumen *root* server dibiarkan lepas (misal saat *development* di Laragon).

**Perencanaan dan Diskusi 403 Forbidden:**
Kami lalu membuat *Implementation Plan* yang berisi pembuatan `.htaccess` yang akan mematikan *Directory Listing* dan melakukan *silent redirect* ke `/public/`.**Modifikasi Halaman 403 Kustom:**

- Kami merancang array baru pada *wordsearch puzzle* yang tadinya menampilkan "404 PAGE NOT FOUND"..
- Teks utamanya diubah menjadi penolakan izin akses, tanpa menghilangkan nama *easter egg.*
- Kami menggandakan aset menjadi `403.css` dan `403.js`, memperpanjang array elemen kelas dari `.fifteen` hingga `.seventeen`.
